# Security Headers (Web)

This document provides a baseline security header policy for the Vite SPA. Adjust hostnames to match your deployment and Supabase project URL.

## Recommended CSP
Use a restrictive CSP without `unsafe-eval` and avoid `unsafe-inline` for scripts. Styles may require `unsafe-inline` due to inline styles generated by UI libraries.

```
Content-Security-Policy: default-src 'self'; base-uri 'self'; object-src 'none'; frame-ancestors 'none'; form-action 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; img-src 'self' data: https:; font-src 'self' data:; connect-src 'self' https://*.supabase.co wss://*.supabase.co; upgrade-insecure-requests; block-all-mixed-content;
```

### Notes
- Replace `https://*.supabase.co` with your project URL if you want tighter scoping.
- If you add analytics or other third‑party APIs, update `connect-src` accordingly.
- For iOS native (Capacitor) builds, CSP applies to the web layer only.

## Additional Headers
```
Strict-Transport-Security: max-age=63072000; includeSubDomains; preload
X-Content-Type-Options: nosniff
Referrer-Policy: strict-origin-when-cross-origin
Permissions-Policy: camera=(), geolocation=(), microphone=(), payment=()
X-Frame-Options: DENY
```
